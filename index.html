Intro

Me the coder 70% / Me the gardener 30%

Schaffe, schaffe Häusle baue - are four German words that sum me up pretty well. They translate to " Work hard, work hard, build a little house" which symbolically stands for the diligence of the Swabian people in the South-West of Germany. Being a Swabian, I pride myself in being reliable and consistent in my work. I'm a system thinker. I first try to establish a pattern in order to find the most effective solution. I love to build (not houses particularly) but systems that endure, increase efficiency and have a lasting effect on others. Thus, in a working environment one would usually find me at the coffee machine trying to figure out how to connect it to the internet to let it run an automated script so nobody has to struggle with an empty coffee jar again.
When I’m not building systems, you can find me with my second passion - plants. My garden is cluttered with herbs and vegetables. The independence of plants to be able to grow by themselves fascinates me. My favorite plants are beans and coriander.

Languages

As every developer’s career starts with HTML, mine did too. HTML is a foundational language and I can confidently claim to have a good understanding of it.

I have a sort of love-hate relationship with CSS. On the one hand, it is essential to any website, but on the other hand there are so many functionalities to learn like CSS animations. I used CSS animation in my introduction section, but getting it where it is now took some time. Despite this, I feel that CSS is a powerful language with a broad spectrum of functionalities that are underappreciated.

Most of the functionality of this website was built using JavaScript. For example, the scrollLock function which actives the moment my title picture is outside the viewport and the custom scroll bar on the left which listens to scrollEvents. It is a lot of fun playing around with Javascript on the front-end but it also powers the back-end of this website by running a node.js server.

Jquery is the Swiss Army knife of Javascript libraries. Again, much of the front-end of this website is based on Jquery code like this text which uses a toggle function to add and remove a class. I personally love Jquery just because it saves me so much time.

Most of the applications under the Portfolio section are written in PHP. For the Umthunzi Farming Community project I used both PHP and Node.js, but I will go into more detail about that in the Portfolio section.

MySQL is the relational database management system I have studied. What amazes me most about MySQL is the performance of retrieving data and executing insertions.

Working with Datomic has teached me some valuable skills one of them being: Datalog. Datomic has a very good datalog integration and works well with Clojure. Coming from a SQL world it is radically different but intriguing.

ClojureScript is my front-end language of choice. I am using it with Reagent which is a ClojureScript library that enables one to do JS-React components using plain Clojure syntax. Creating beautiful front-end applications with little effort is REAL!

As being functional I can safely say that Clojure was one of the most challenging languages I have learned so far. However, once internatlised the simplcity of doing things has made Clojure my go to language.

Portfolio

Spellgeek was my first serious entrepreneurial venture which I founded with a good friend from Stellenbosch University. The website offers academic proofreading and editing services to foreign language speakers. I contributed to the layout, design and code of the website. The business is currently on hold while we continue to raise capital.

The To-Do-App is a PHP application that includes a signup page, a login page, and the app itself. The entire signup and login process is up-to-date with the highest security standards. The app writes all the user’s to do list entries to a database. When a task is done it will strike through the item, marking it as done. The user is still able to see their entries when they next login.

The IOT world has taking me by storm. As such I have started a little project myself called Demeter written in Python. The project involves an ESP8266 mircocontroller, relays and different sensors that measure the environment. Demeter is meant for a greenhouse or grow tent and can be controlled from the internet. I have incoperated google charts to represent data that is constantly written to the database. This project is only in its infant shoes so watch this space!

The Hotel Booking Form was my first introduction to MySQL in conjunction with PHP. The application allows the user to input their first and last name, choose a check in and check out date, and finally, select one of four hotel options. The booking information is summarized and displayed back to the user. The booking is only written to the database when the user confirms the information. If a duplicate booking exists, the user is notified. Otherwise, the booking gets confirmed.
